## 五层文学支撑体系

| 文件 | 管什么 | 比喻 |
|------|--------|------|
| `philosophy.md` | 思想骨架 | 这本书在追问什么 |
| `characters.md` | 人物血肉 | 谁在追问 |
| `character_archetypes/` | 人格原型（通用层） | 追问者的底层人格池 |
| `motifs.md` | 意象肌理 | 追问的姿态长什么样 |
| `world.md` + `world_texture.md` | 冲突舞台 + 世界触感 | 棋盘 + 棋盘摸起来是木头还是石头 |
| `technique_library/` | 写作技巧（通用层） | 怎么追问 |

## 关键设计决策（经十轮外部 AI 审查）

1. **RAG 使用本地 QMD 引擎**（BM25 + 向量 + reranking，内置 embedding，零外部依赖）
2. **质量门禁 6 步**：记忆同步→一致性→哲学对齐（人工反思表单）→风格→反AI→门禁判定
3. **数据闭环克制化**：主观评分 1-5 + 趋势展示，不做自动归因，50 章后才启用统计规则
4. **批量写作 N ≤ 3**，每章间强制人工确认
5. **哲学对齐 = 人工反思表单**（3 个问题），不是自动检查
6. **趋势监控手动化**（每周 15 分钟），不用 OCR 自动采集
7. **风格锚点每 50 章提示更新**，尊重作者自然成长
8. **世界质感分两层**：底色质感（全书不变）+ 动态质感快照（随剧情变化）
9. **/填充配置 命令**：用户碎片感性输入 → AI 结构化填充（含反平均化提醒）
10. **成本控制**：单章 ~18,800 token，全年 ~$80-$100

## 执行路径

- Phase 0：定位确认 + 技术探针 + 配置填充 + 构思大纲（当前）
- Phase 1：环境搭建（安装 skill + 技巧库 + 原型库读取逻辑）
- Phase 2：引擎开发（记忆+门禁+反AI+RAG）
- Phase 3：数据闭环（自评+录入+趋势展示+人工复盘）
- Phase 4：趋势系统（手动记录模板）
- Phase 5：试写验证
