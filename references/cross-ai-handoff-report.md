# 跨 AI 交接报告模板

> 当 Hermes 生成上下文报告供 Claude Code（或其他 AI）参考时使用。

## 使用场景

- Hermes 开发完功能，需要 Claude Code 做代码改造
- 项目迁移到新仓库，需要下游 AI 了解现状
- 多 AI 协作时，一方积累的上下文需要传递给另一方

## 报告结构

```markdown
# <项目名> — <报告主题>

> 生成者：Hermes Agent
> 日期：YYYY-MM-DD
> 目的：为 [目标AI] 提供 [具体信息]

## 一、当前状态
- 关键数据点（文件数、脚本数、大小）
- Git 状态（分支、最新提交）
- 依赖关系

## 二、关键发现
- 不该在 git 中的文件（运行时数据、敏感信息）
- 版本不一致
- 路径失效
- 架构问题

## 三、资产清单
- 按类别列出文件数量和用途
- 区分"代码"和"数据"

## 四、改进建议
- 必做项（阻塞性问题）
- 建议项（质量改进）
- 可选项（清理/归档）

## 五、分工说明
- 各 AI 负责什么
- 同步机制是什么
```

## 关键原则

1. **信息不对称是常态**——另一方不知道本地环境细节、QMD 索引、pitfall 等
2. **用数据说话**——文件数量、大小、路径都要实际核实，不靠记忆
3. **区分"代码"和"数据"**——这是跨 AI 协作的核心边界
4. **隐私扫描前置**——推 GitHub 前主动检查 API key、邮箱、本地路径
