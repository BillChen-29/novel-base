## 三级知识过滤

AI_NovelGenerator 的 knowledge_filter_prompt 实现了三级过滤：

### 1. 冲突检测
- 删除与已有摘要重复度 >40% 的内容
- 标记存在世界观矛盾的内容（使用 ▲ 前缀）

### 2. 价值评估
- **关键价值点（❗）**：
  - 提供新的角色关系可能性
  - 包含可转化的隐喻素材
  - 存在至少2个可延伸的细节锚点
- **次级价值点（·）**：
  - 补充环境细节
  - 提供技术/流程描述

### 3. 结构重组
按四个维度分类：
- 情节燃料（可用于推进剧情）
- 人物维度（可用于角色发展）
- 世界碎片（可用于世界观丰富）
- 叙事技法（可用于写作技巧提升）

每个分类添加适用场景提示（如"可用于XX类型伏笔"）。

## 基于章节距离的内容使用规则

AI_NovelGenerator 的 apply_content_rules：

| 距离 | 处理方式 | 原因 |
|------|---------|------|
| ≤2章 | 跳过或降权 | 避免重复引用刚写的内容 |
| 3-5章 | 修改≥40%才能用 | 适度改写，保持新鲜感 |
| >5章 | 可引用核心设定 | 距离够远，可以复用 |

**实现思路**：在 plot_rag 搜索结果中加入 `chapter_distance` 字段，unified_search() 根据距离调整权重。

## 章节元数据结构

AI_NovelGenerator 的 chapter_info 结构比我们的 beat/outline 更丰富：

```python
chapter_info = {
    'chapter_role': '转折章节',        # 开场/铺垫/转折/高潮/收尾
    'chapter_purpose': '揭示真相',     # 核心叙事功能
    'suspense_level': '高',            # 悬念密度
    'foreshadowing': '回收第3章伏笔',  # 伏笔操作
    'plot_twist_level': '★★★★☆',     # 认知颠覆程度
    'characters_involved': '...',      # 涉及角色
    'key_items': '...',                # 关键道具
    'scene_location': '...',           # 场景位置
}
```

**与我们的系统对比**：
- 我们的 beat/outline 只有"发生什么"
- 他们的 chapter_info 还有"这章的叙事功能是什么"
- 叙事功能标签对 RAG 搜索有帮助（搜"转折章节"能找到所有转折点）

## 搜索关键词自动生成

AI_NovelGenerator 用 LLM 从章节元数据自动生成搜索关键词：

```python
# 他们的 knowledge_search_prompt
# 输入：章节元数据（角色、道具、场景、伏笔等）
# 输出：3-5组检索词，用"·"连接
# 示例：科技公司·数据泄露
#       地下实验室·基因编辑·禁忌实验
```

**与我们的对比**：
- 我们直接传原始 query 给 QMD
- 他们先用 LLM 生成结构化检索词
- 他们的检索词更精准，噪音更少

| 改进项 | 优先级 | 工作量 | 说明 |
