# 知乎盐选字体加密分析

> 日期：2026-05-09
> 状态：已验证，无自动化解法（动态加密）

## 核心机制

知乎盐选使用**动态字体加密**（`dynamic_font_schema: 1`）：
- 每次请求生成 4 个自定义 TTF 字体（base64 内嵌在 CSS `@font-face` 中）
- 两个字体家族：小字体（6个高频字）+ 大字体（80个常用字+标点）
- HTML 中的 Unicode 字符是加密的，字体渲染后才是正确的视觉文字
- **每次页面加载映射不同** — 无法复用映射表

## 已验证的失败方案

| 方案 | 结果 | 原因 |
|------|------|------|
| textContent / DOMParser | ❌ | HTML 本身就是加密 Unicode |
| Selection API | ❌ | 返回加密 Unicode |
| 去掉自定义字体 | ❌ | HTML 文本不变 |
| 剪贴板复制 | ❌ | 知乎禁用了 copy 事件 |
| 字形尺寸/哈希对比 | ❌ | 三个字体字形完全不同 |
| screencapture（沙箱） | ❌ | 沙箱内无法截图 |
| Playwright 无头浏览器 | ❌ | 知乎反爬检测 |
| zhihu_tools marketSpider | ❌ | API 返回 HTML 而非 JSON |

## 可行方案

1. **用户手动下载** — 最可靠，用户在浏览器中复制/导出文本
2. **Chrome 远程调试 + 截图 OCR** — 需要在宿主机运行（非沙箱）
3. **同页内映射一致** — 如果能获取加密文本，同一页内映射是固定的，可手动解码
