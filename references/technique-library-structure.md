# 技法库结构设计

> 2026-05-07 讨论结论

## 问题

16个技法文件是扁平结构，缺乏使用场景上下文。搜索"悬念"只能找到 `suspense-laddering.md`，不知道它适合什么场景。

## 三层结构方案

### 第一层：原子概念（已有，底层原理）
- suspense.md — "悬念是什么，核心原理"
- twist-ending.md — "反转结尾是什么"
- horror-atmosphere.md — "恐怖氛围是什么"

### 第二层：场景技法（已建，`_scenarios/` 目录）
- opening.md — 开场：冷开场+角色引入+世界观暗示
- climax.md — 高潮：节奏加速+情感爆发+爽点释放
- cliffhanger.md — 章末悬念：钩子+信息差+问号
- character-intro.md — 角色引入
- world-building.md — 世界观展示
- fight-scene.md — 战斗场景
- dialogue-tension.md — 对话张力
- revelation.md — 揭秘/反转
- romance.md — 感情线
- training.md — 修炼/升级

### 第三层：作家参考（可选，`_writers/` 目录）
- faulkner.md — 福克纳：多视角+意识流+空间叙事
- calvino.md — 卡尔维诺：元小说+轻逸+晶体模式
- poe.md — 爱伦·坡：悬念阶梯+恐怖氛围
- o-henry.md — 欧·亨利：反转结尾
- twain.md — 马克·吐温：幽默+讽刺

## 设计决策

1. **场景层是核心**：写作时AI的问题是"我正在写这种场景，该用哪些技法"，不是"悬念是什么"
2. **场景文件交叉引用四层数据库**：每个场景文件链接相关母题、角色、技法、风格、节奏
3. **场景层不是新建库，是建索引**：已有数据库齐全，缺的是串联入口
4. **作家层待定**：风格库已有6本网文风格档案，文学作家的技法组合可以后续补充

## 写作时的调用流程

```
novel_chapter_writer.py 写第5章
  ↓
大纲说：本章是"主角首次战斗，章末发现身世秘密"
  ↓
场景判断：fight-scene + revelation + cliffhanger
  ↓
加载对应场景文件，提取技法建议
  ↓
结合风格库（如天渊风格：腹黑幽默+众人震惊）
  ↓
生成章节
```
