# 场景：揭秘/反转（Revelation）

## 一句话定义
隐藏的真相被揭示，改变读者对剧情的理解。

## 关联母题（揭秘类型）
- identity-mask（身份面具）— 身份揭示
- forbidden-knowledge（禁忌知识）— 秘密揭示
- prophecy-defied（预言被违背）— 命运揭示
- disguised-predator（伪装的掠食者）— 真面目揭示

## 关联角色（揭秘相关）
- hypocritical-white-moonlight（伪善白月光）— 真面目揭示
- reborn-avenger（重生复仇者）— 身份揭示
- trickster（骗子）— 计划揭示

## 关联技法（怎么写）
- twist-ending.md（反转结尾）— 结构性反转
- suspense-laddering.md（悬念阶梯）— 揭秘前的悬念铺设
- multi-perspective-narration.md（多角度叙述）— 多视角揭示真相

## 关联风格（什么语感）
- 诸神愚戏风格："骗中骗"叙事，读者永远不确定真假
- 游戏入侵风格：重生者信息差，精确复仇

## 写作要点
1. 揭秘前必须有伏笔（至少3章前埋下）
2. 揭秘要有冲击力：不要平淡地"原来如此"
3. 揭秘后要有后果：角色关系/剧情走向发生变化
4. 不要一次揭示太多，留一些悬念给后续章节
